# 그래픽 스타일 가이드 리뷰

> 일시: 2026-02-18
> 참여: designer, developer, reviewer
> 대상: GDD 3절 (그래픽 스타일 가이드), 11절 (에셋 리스트)

---

## 1. Designer 분석

### a) "피 묻은 동화책" 컨셉 구체화

**귀여움 vs 그로테스크 비율: 7:3 권장**

GDD에서 "첫인상은 귀엽고 예쁘지만, 자세히 보면 소름끼치는 디테일"이라고 명시. 귀여움이 "기본값"이고 그로테스크가 "발견 요소"인 구조. 7:3이 컨셉에 정확히 부합. 6:4이면 "그냥 그로테스크한 게임"으로 인식될 위험, 8:2이면 차별화 약화.

**레퍼런스 포지셔닝: Cult of the Lamb > Don't Starve > Binding of Isaac**

뱀서라이크는 캐주얼 타겟이 넓으므로 Cult of the Lamb의 "귀여운 껍데기" 전략이 가장 적합. Binding of Isaac 수준의 그로테스크는 타겟층을 좁힘.

**320x180에서 "소름끼치는 디테일" 표현**

320x180에서 정적 디테일은 사실상 불가능. 32x32 캐릭터가 화면에서 약 18%. 전략 전환 필요:

1. 애니메이션 기반 발견: idle에서 3~4초에 한 번 눈동자 따로 움직임
2. 컬러 불일치: 파스텔 톤 형태 + 과채도 핏빛/독색
3. 환경 이벤트: 배경 창문 뒤 그림자 간헐적 움직임 (타일 애니메이션)
4. 정적 디테일은 UI/메뉴에서: 64x64 초상화, 기억의 서재 삽화

### b) 색상 팔레트 평가

**가독성 문제:**

- 기본 배경 톤 4색 모두 명도가 매우 낮음 (#2D1B3D, #1A3A2A, #5C2D6E, #3D2415)
- 짙은 보라(#2D1B3D)와 자주색(#5C2D6E)의 명도 차이가 작아 인접 시 구분 어려움
- "그림자 고양이"는 어두운 배경에서 안 보일 수 있음
- 16x16 적은 크기가 작아 배경에 묻힐 위험

**개선안:**
1. 중간 명도 색 1개 추가 (#4A3860) → 바닥 타일에 활용
2. 적 스프라이트에 1px 밝은 아웃라인 또는 발밑 그림자 원 표준 적용
3. 어두운 적은 눈이나 특정 부위에 강조색 필수 (형광 초록 눈 등)

**엘리트 오라색 이슈:**
- #C850C0(보라 핑크)이 배경 자주색(#5C2D6E)과 톤 비슷 → 배경 위에서 오라 약해 보임
- 더 밝게 조정하거나 펄스 애니메이션 필수

### c) 캐릭터/적 디자인

**32x32 vs 16x16 크기 비율:** 적절. 2:1 비율이 플레이어 위치 인식 보장.

주의점:
- 거울 유령(24x24): 일반 적보다 크고 플레이어보다 작은 애매한 크기. 엘리트(1.5배=24x24)와 동일 크기 → 혼동 가능성
- 2등신 권장: 32px에서 3등신은 머리가 너무 작아 표정/특징 전달 불가

**보스 시각:**
- 보스 2(96x96)는 320x180에서 가로 30%, 세로 53% 차지
- 보스 뒤에 숨은 적이 보이지 않는 문제 → 보스 등장 시 일반 적 스폰 줄이기

### d) UI 디자인

**폰트 리스크:**
- 한글 손글씨 + 저해상도 = 가독성 붕괴
- 해결: 2중 폰트 (제목=손글씨, 본문=깔끔한 폰트)

**HUD 점유율:**
- 상단 HP바/타이머/킬/골드: ~16px (9%)
- XP바: ~8px (4%)
- 무기 슬롯 6개: ~16px (9%)
- 하단 패시브+미니맵: ~16px (9%)
- 총 ~56px/180px = **31%** → Vampire Survivors는 15~20%

**레벨업 카드:**
- 3장: 80x3=240px+간격40px, 적절
- 4장: 80x4=320px, 간격 없음, 불가능 → 3장 고정

### e) 차별화 포인트

현재 뱀서라이크 시장에서 "피 묻은 동화책"은 고유 포지션:
1. 낮/밤 대비 (어떤 뱀서라이크에도 없음)
2. 적 사망 시 하트/별 파티클이지만 핏빛 (바이럴 포인트)
3. 매 런마다 단서 수집하는 스토리 구조

MVP 차별화 필수 장면 3가지:
1. 타이틀 화면 낮→밤 전환
2. 적 사망 시 귀여운 형태+잔혹한 색상 파티클
3. 결과 화면 동화책 UI

## 2. Developer 분석

### a) 320x180 픽셀 퍼펙트 제약

**16x16 디테일:** 256px에서 이빨, 촉수, 균열 표현은 가능하지만 제한적. 실루엣으로 귀여움, 2~3px 디테일로 기묘함 암시.

**필수 설정:**
- `display/window/stretch/mode = "canvas_items"`
- `display/window/stretch/aspect = "keep"`
- `display/window/stretch/scale_mode = "integer"`
- `snap_2d_transforms_to_pixel = true`
- `snap_2d_vertices_to_pixel = true`

**integer 스케일링:** 1080p(x6), 1440p(x8)은 딱 맞음. 1366x768은 x4=1280x720으로 스케일, 나머지 레터박스.

**서브픽셀:** 물리 위치는 float, 렌더링 위치는 snapped로 스냅. 파티클/데미지 숫자도 전부 스냅 권장.

### b) 스프라이트 & 애니메이션

**크기별 표현 수준:**

| 크기 | 픽셀 수 | 표현 수준 | 적용 대상 |
|------|---------|----------|----------|
| 8x8 | 64px | 아이콘 수준, 형태만 인식 | 보석, 드롭, 파티클 |
| 16x16 | 256px | 실루엣 + 1~2개 핵심 디테일 | 일반 적, 타일, 발사체 |
| 24x24 | 576px | 실루엣 + 3~4개 디테일, 표정 가능 | 중형 적 |
| 32x32 | 1024px | 캐릭터성 완전 표현 | 플레이어, 무기 아이콘 |
| 64x64 | 4096px | 복잡한 디테일 (관절, 장식) | 보스 1, 초상화 |
| 96x96 | 9216px | 풍부한 디테일 | 보스 2 |

**메모리:** 전체 스프라이트 비압축 2~3MB. GPU 텍스처로도 문제없음.

**아틀라스 전략:**
- `enemies_atlas.png`: 모든 일반 적 + 엘리트 (1 draw call)
- `player_atlas.png`: 플레이어 캐릭터별
- `effects_atlas.png`: 파티클, 데미지 숫자, 이펙트
- `ui_atlas.png`: HUD, 아이콘
- 보스는 크기가 크므로 개별 시트

### c) 성능과 비주얼 균형

**500마리+ 적:**
- 엘리트 오라: 동시 5~10마리, 개별 셰이더 처리 가능
- 사망 파티클: CPUParticles2D 풀링 (30~50개). pool_manager에서 관리.
- GPUParticles2D는 보스 전용

**MultiMeshInstance2D:**
- `instance_color`로 엘리트 틴트, `custom_data`로 애니메이션 프레임/피격 플래시/상태 플래그
- MVP: 적 종류별 별도 MultiMesh (통합은 풀 릴리즈)

**GPUParticles2D vs CPUParticles2D:**
- GPU: 수천~만 개, 셰이더 기반, 보스 이펙트/배경 분위기용
- CPU: 수십~수백 개, 풀링 쉬움, 적 사망/피격 이펙트/드롭 반짝임용

**데미지 숫자 20개:** 비트맵 폰트 + Tween. 성능 영향 거의 없음.

### d) 셰이더 활용

**엘리트/보스 오라:** 외곽선 셰이더 10줄 내외. UV 상하좌우 1텍셀 샘플링. 펄스는 sin(TIME) 기반. 5~10개 엘리트에만 적용.

**밤 분위기 MVP 권장:**
- CanvasModulate: 전체 화면 색조 (1줄 설정)
- PointLight2D: 플레이어 주변 + 횃불 3~5개
- 커스텀 셰이더: 풀 릴리즈에서 추가

**피격/상태 셰이더:**
- 피격 플래시: `mix(original_color, white, flash_amount)`, 0.1초
- 독: 초록 틴트 + sin(TIME) 맥동
- 500마리 동시 적용 가능. `material.resource_local_to_scene = true` 필요.

**안개 (스테이지 2):** 반투명 스프라이트 2~3장 스크롤 (MVP). Perlin noise 셰이더는 풀 릴리즈.

### e) 타일맵 & 환경

**TileMap 레이어:**
- Layer 0: 바닥 (돌바닥, 흙, 풀)
- Layer 1: 벽/장애물 (충돌 있음)
- Layer 2: 장식 (충돌 없음, 오버레이)

**파괴 오브젝트:** 별도 Area2D 노드 권장. 애니메이션, 파티클, 드롭 로직에 유리. 최대 240개 노드.

**패럴랙스:**
- Layer -2: 먼 배경 (밤하늘, 달) - 비율 0.1~0.2
- Layer -1: 중간 배경 (건물 실루엣) - 비율 0.3~0.5
- Layer 0: TileMap - 비율 1.0
- Layer 1: 전경 오버레이 (안개, 넝쿨) - 비율 1.1~1.2

### f) MVP vs 풀 릴리즈

**MVP 단순화:**

| 기능 | MVP | 이유 |
|------|-----|------|
| 밤 분위기 | CanvasModulate + PointLight2D 1개 | Light2D 다수는 최적화 필요 |
| 적 사망 | CPUParticles2D 풀링 | GPU 커스텀은 시간 소모 |
| 엘리트 오라 | 단순 외곽선 셰이더 1px | 복잡한 글로우는 풀 릴리즈 |
| 보스 이펙트 | 원형 스프라이트 애니메이션 | 셰이더 기반은 후순위 |
| 안개 | 반투명 스프라이트 2장 스크롤 | Perlin noise는 풀 릴리즈 |
| 타이틀 전환 | 정적 2장 크로스페이드 | 실시간 전환은 후순위 |
| MultiMesh | 종류별 별도 | 통합 아틀라스+UV는 복잡 |
| 패럴랙스 | 1~2 레이어 | 다중 레이어는 후순위 |

**풀 릴리즈 추가:**
1. 고급 조명 (다수 PointLight2D + Shadow 2D)
2. 후처리 셰이더 (비네팅, 색수차, CRT)
3. 날씨 시스템 (GPUParticles2D 기반 비/눈/번개)
4. 보스 전용 연출 (화면 왜곡, 컬러 시프트)
5. 통합 MultiMesh + UV 셰이더
6. Perlin noise 안개
7. 물 반사/왜곡 셰이더
8. 잔상(Afterimage) 효과
9. 적 유형별 고유 사망 파티클
10. 패럴랙스 배경 애니메이션

## 3. Reviewer 종합 리뷰

### 기획-기술 간 갭 (3건)

1. **일반 적 아웃라인 + MultiMesh 충돌:** 500마리+ 시 MultiMesh는 개별 셰이더 불가. 해결 = 셰이더 아웃라인이 아닌 스프라이트 자체에 1px 아웃라인 내장 (에셋 단계 처리)
2. **보스 2(96x96) 화면 53% 점유:** Designer의 "반투명" 제안은 위협감 감소 → 기각. spawn_manager에서 보스 등장 시 일반 적 스폰 감소로 대응
3. **엘리트 오라색 #C850C0 vs 배경 #5C2D6E 톤 충돌:** #FF6BD6(밝은 자홍)으로 변경

### MVP 현실성 (KISS/YAGNI)

- 제거: 미니맵 (적 인디케이터로 충분)
- 수정: 레벨업 카드 3장 고정
- 단순화: 거울 유령 미러링 AI → 단순 추적+벽통과
- 유지: 낮→밤 전환, 귀여운+잔혹 파티클, 동화책 UI (차별화 핵심)

### CLAUDE.md 원칙 준수

- 관심사 분리(6절): 아틀라스 4분류 적절. 환경 연출 매니저 미배치 → 후속 논의 필요
- 데이터 관리(8.4절): 적 시각 속성을 EnemyVisualData Resource로 분리하면 OCP 완전 부합
- 성능 규칙(8.6절): CPUParticles2D 풀링, 종류별 MultiMesh 등 규칙 부합
